## Responsibilities

Maintainers are responsible for:

- Reviewing and merging pull requests
- Triaging and responding to issues
- Making decisions about the project direction
- Ensuring code quality and test coverage
- Managing releases and versioning
- Enforcing the Code of Conduct
- Responding to security issues

## Becoming a Maintainer

Maintainers are chosen based on their contributions to the project and demonstrated commitment to its success. If you're interested in becoming a maintainer:

1. Make consistent, high-quality contributions to the project
2. Help review pull requests and triage issues
3. Engage positively with the community
4. Demonstrate deep understanding of the project's goals and architecture

Current maintainers will reach out to active contributors when appropriate.

## Maintainer Guidelines

### Reviewing Pull Requests

- Be respectful and constructive in your feedback
- Ensure code follows the project's style guidelines
- Verify that tests pass and coverage is maintained
- Check that documentation is updated when necessary
- Request changes or approve within a reasonable timeframe

### Handling Issues

- Respond to new issues within 48 hours when possible
- Add appropriate labels to help categorize and prioritize
- Ask for clarification when needed
- Close duplicate or resolved issues promptly

### Making Decisions

- Major decisions should be discussed with other maintainers
- Use GitHub discussions for architectural changes
- Document important decisions in the repository
- Consider community feedback for significant changes

### Release Process

- Follow semantic versioning guidelines
- Update CHANGELOG.md with all changes
- Run full test suite before releasing
- Tag releases appropriately
- Update documentation as needed
